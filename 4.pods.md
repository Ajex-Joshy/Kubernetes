## Pods

- A pod is a smallest deployable unit in kubernetes
- A pod contains one or more containers that run together on the same node

```
Node
 └── Pod
      ├── Container (App)
      └── Container (Sidecar)

```

- pods run on worker nodes
- pods share:
  - network
  - IP address
  - storage volume
- pods are ephemeral(temporary)

if a pod dies, kubernetes (controller) creates a new one

**why pods exists?**
because containers needs an environment to run inside kubernetes

**Pod characterstics**

- All containers inside a pod share
  - same IP address
  - same port space
  - shared volume

**we can create a pod in Imperative and Declarative way**

### Imperative way to create a pod

- we will tell k8s step by step commands to create a pod

```
kubectl run nginx-pod --image=nginx
```

This command immediately creates a pod

### Declarative Method

- Here we declare the desired state of the system using YAML file, and k8s ensures this state exists

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx-pod
spec:
  containers:
    - name: nginx
      image: nginx
      ports:
        - containerPort: 80
```

`kubectl apply -f pod.yaml`

Kubernetes checks the desired state in YAML and makes the cluster match it.

### Pod Lifecycle

| Phase     | Meaning                        |
| --------- | ------------------------------ |
| Pending   | Pod accepted but not scheduled |
| Running   | Pod running successfully       |
| Succeeded | Pod completed successfully     |
| Failed    | Pod terminated with failure    |
| Unknown   | Node communication issue       |
