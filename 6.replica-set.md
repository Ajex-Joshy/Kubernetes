## ReplicaSet

A **ReplicaSet** in Kubernetes ensures that a specified number of identical **Pods** are always running in the cluster.

Its main responsibility is to maintain a **stable set of replica Pods** at any given time.

If a Pod:

- crashes
- gets deleted
- fails

The ReplicaSet will automatically **create a new Pod** to maintain the desired number of replicas.

Example:

Desired replicas = 3

```
ReplicaSet
├── Pod 1
├── Pod 2
└── Pod 3
```

If one Pod fails, the ReplicaSet immediately creates a new one to keep the count at 3.

---

### Important Components of a ReplicaSet

1. **replicas**  
   Defines how many Pods should be running.

```
replicas: 3
```

2. **selector**  
   Identifies which Pods belong to the ReplicaSet using labels.

```
selector:
  matchLabels:
    app: nginx
```

3. **template**  
   Defines the Pod configuration used to create new Pods.

```
template:
  metadata:
    labels:
      app: nginx
  spec:
    containers:
      - name: nginx
        image: nginx
```

---

### Ways to Manage Replicas

1. **Declarative (YAML)**

Define the desired state in a YAML file.

```
kubectl apply -f replicaset.yaml
```

2. **Imperative (Command)**

Directly modify the replica count using commands.

```
kubectl scale rs nginx-rs --replicas=5
```

3. **Live Editing**

Edit the running resource configuration directly.

```
kubectl edit rs nginx-rs
```

---

### Important Note

In real-world Kubernetes usage, developers rarely create ReplicaSets directly.

Instead the common structure is:

Deployment → ReplicaSet → Pods

A **Deployment** automatically manages ReplicaSets and provides features like rolling updates and rollbacks.

---

### Useful ReplicaSet Commands

1. **View ReplicaSets in the cluster**

```
kubectl get rs
```

Shows all ReplicaSets along with desired, current, and ready pod counts.

2. **View Pods created by the ReplicaSet**

```
kubectl get pods
```

Displays the Pods that are managed by the ReplicaSet.

3. **Describe a ReplicaSet**

```
kubectl describe rs nginx-rs
```

Shows detailed information including events, selectors, and the Pod template.

4. **Scale a ReplicaSet**

```
kubectl scale rs nginx-rs --replicas=5
```

Changes the number of desired Pods managed by the ReplicaSet.

5. **Delete a ReplicaSet**

```
kubectl delete rs nginx-rs
```

Deletes the ReplicaSet and the Pods it manages.

6. **Watch Pods in real time**

```
kubectl get pods -w
```

Continuously watches Pod state changes such as `Pending`, `Running`, or `Terminating`.
